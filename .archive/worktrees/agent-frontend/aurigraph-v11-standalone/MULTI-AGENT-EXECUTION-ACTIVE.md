## üö® BLOCKER ESCALATION PROTOCOL

### Level 1: YELLOW ALERT (24-hour resolution)
**Severity**: Single component affected, workaround available, no schedule impact

**Example**: "TypeScript compilation error in one component"

**Resolution**:
1. Reported in daily standup
2. Peer assistance offered
3. Resolved within 24 hours
4. Document root cause

**Owner**: Relevant agent

### Level 2: ORANGE ALERT (4-hour resolution)
**Severity**: Multiple components affected, no workaround, potential schedule impact

**Example**: "Mock API endpoint not returning expected data"

**Resolution**:
1. Escalated immediately
2. DDA + FDA collaborate
3. 4-hour target resolution
4. Implement workaround if needed
5. Root cause analysis

**Owner**: DDA + FDA + PM

### Level 3: RED ALERT (Immediate response)
**Severity**: Sprint-level impact, blocking multiple developers, schedule at risk

**Example**: "GitHub branch protection preventing all merges"

**Resolution**:
1. Emergency escalation
2. All agents mobilized
3. Immediate investigation
4. Executive decision on path forward
5. Post-incident review

**Owner**: Project Manager + all agents

### Escalation Contacts
- **YELLOW**: Slack standup, handle within team
- **ORANGE**: Slack + immediate huddle
- **RED**: Emergency call, Project Manager + team
