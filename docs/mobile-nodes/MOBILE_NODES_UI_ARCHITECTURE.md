# Mobile Nodes UI Architecture

## V12 Mobile Nodes UI Design Document

**Version**: 1.0.0
**Date**: November 29, 2025
**Author**: Backend Development Agent (BDA)
**Status**: Architecture Design Complete

---

## 1. Overview

The Mobile Nodes UI provides a responsive, native-like interface for managing and monitoring lightweight blockchain nodes on mobile devices. This architecture supports both iOS and Android platforms through React Native with shared TypeScript codebase.

### 1.1 Key Features

- **Node Management**: Start/stop/configure lightweight nodes
- **Real-time Monitoring**: Live metrics via gRPC-Web streaming
- **Transaction Submission**: Send transactions from mobile
- **Validator Delegation**: Delegate stake to validators
- **Notification System**: Push notifications for node events
- **Offline Support**: Cached data and offline transaction queuing

### 1.2 Target Platforms

| Platform | Framework | Minimum Version |
|----------|-----------|-----------------|
| iOS | React Native | iOS 14.0+ |
| Android | React Native | Android 8.0 (API 26)+ |
| Web (PWA) | React | Chrome 90+, Safari 14+ |

## 5. API Integration

### 5.1 gRPC Streaming Endpoints

| Stream | Endpoint | Update Interval | Description |
|--------|----------|-----------------|-------------|
| Metrics | `StreamMetrics` | 1s | Real-time TPS, latency |
| Validators | `StreamValidatorEvents` | 2s | Validator status updates |
| Consensus | `StreamConsensusEvents` | 500ms | Consensus state changes |
| Network | `StreamNetworkEvents` | 3s | Network topology |
| Transactions | `StreamChannelTransactions` | 500ms | New transactions |

### 5.2 REST API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/v11/nodes` | List user's nodes |
| POST | `/api/v11/nodes` | Create new node |
| GET | `/api/v11/nodes/{id}` | Get node details |
| PUT | `/api/v11/nodes/{id}/start` | Start node |
| PUT | `/api/v11/nodes/{id}/stop` | Stop node |
| POST | `/api/v11/transactions` | Submit transaction |
| GET | `/api/v11/wallet/balance` | Get wallet balance |

## 6. Security Considerations

### 6.1 Key Management

- Private keys stored in iOS Keychain / Android Keystore
- Biometric authentication required for signing
- Hardware-backed encryption where available
- Key derivation using BIP-39/BIP-44

### 6.2 Network Security

- TLS 1.3 for all API communication
- Certificate pinning for production
- JWT tokens with short expiry (15 min)
- Refresh tokens stored securely

### 6.3 Data Protection

- Sensitive data encrypted at rest
- Clear memory after use
- No sensitive data in logs
- Secure clipboard handling
