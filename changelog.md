## Development Guidelines

### Version Numbering
- **Major version** (X.0.0): Revolutionary features, breaking changes
- **Minor version** (X.Y.0): New features, backward compatible
- **Patch version** (X.Y.Z): Bug fixes, security patches

### Change Categories
- **Added**: New features
- **Changed**: Changes in existing functionality
- **Deprecated**: Soon-to-be removed features
- **Removed**: Removed features
- **Fixed**: Bug fixes
- **Security**: Security improvements

### JIRA Integration
All changes should reference relevant JIRA tickets:
- Format: `([TICKET-ID](https://aurigraphdlt.atlassian.net/browse/TICKET-ID))`
- Example: `([AV11-8](https://aurigraphdlt.atlassian.net/browse/AV11-8))`

### Maintenance Notes
- Update this changelog for every significant change
- Include JIRA ticket references for traceability
- Follow semantic versioning for all releases
- Document breaking changes with migration guides
- Include performance impact notes for major changes
